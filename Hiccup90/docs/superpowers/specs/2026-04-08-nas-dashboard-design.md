# NAS 导航页设计

## 目标

构建一个适合 NAS 自托管场景的导航页，满足以下方向：

- 配置自由度接近 `gethomepage/homepage`：分组、链接卡片、标签、搜索、布局和主题可配置。
- 配置体验接近 `sun-panel`：不依赖后端，直接在前端界面完成新增、编辑、删除、排序、导入导出。
- 视觉风格接近 `sun-panel` / `dashdot`：现代、克制、强调空间和材质感。
- 轻量高性能：无框架、无第三方依赖、静态资源可直接部署到 NAS。

## 方案

采用无依赖静态 SPA：

- `index.html` 提供应用骨架。
- 原生 ES Modules 拆分核心逻辑和 UI 渲染。
- `localStorage` 持久化配置。
- JSON 导入导出作为配置交换格式。
- 原生 Drag and Drop 处理分组和卡片排序。

## 信息架构

页面分为四层：

1. 左侧品牌与全局操作区：站点标题、搜索、编辑模式、设置入口。
2. 顶部工作区摘要：当前时间、服务数量、分组数量、搜索引擎入口。
3. 主内容区：可折叠服务分组，每组下为服务卡片。
4. 设置抽屉：主题、布局、搜索引擎、分组管理、链接管理、导入导出。

## 数据模型

配置以单一 JSON 对象保存：

- `app`: 标题、副标题、搜索引擎、布局密度、主题参数。
- `groups[]`: 分组元信息、排序、是否折叠。
- `items[]`: 每个服务的名称、URL、描述、标签、图标、状态色、所属分组。
- `widgets[]`: 首屏摘要小组件配置，第一版提供时钟、统计、便签。

所有实体带稳定 `id`，支持未来迁移。

## 交互

- 默认浏览模式：快速搜索、打开服务、折叠分组。
- 编辑模式：显示新增、编辑、删除、拖拽句柄。
- 设置抽屉内支持：
  - 修改站点标题、主题色、背景风格、卡片密度
  - 新增/编辑/删除分组
  - 新增/编辑/删除服务链接
  - JSON 导出、粘贴导入、恢复默认

## 视觉方向

视觉基调：冷静、偏专业的 NAS 控制台，不做厚重后台卡片墙。

- 深浅两套主题，默认深色石墨蓝。
- 背景使用低对比渐变和微弱网格噪声，避免纯色平铺。
- 卡片采用低边框、高留白、轻磨砂和柔和投影。
- 强调大标题、精炼文案和明确层级。

## 测试与验证

先为以下核心逻辑写测试：

- 配置归一化与迁移
- 搜索匹配
- 分组内与跨分组重排

完成后再做页面实现，最后运行：

- 单元测试
- 语法检查
- 浏览器手动验收所需的本地静态预览

## 已知边界

- 第一版不接入后端，不做服务探活、监控面板或容器 API 集成。
- 第一版使用 JSON 配置，不做 YAML 兼容解析，以保持零依赖和高性能。
