# 万事录 私人管家设计说明

## 产品定位

万事录是一款以聊天为入口的私人管家应用。它把碎片记录、个人偏好、主题知识、长期档案和外部聊天消息收拢到一个浅色自然风格的生活中枢里，让用户既能像和助理聊天一样输入需求，也能持续沉淀自己的长期记忆。

首版不追求复杂多端同步或真实第三方接入，而是优先把四件事做扎实：

- 管家式聊天入口
- 分层分类持久化记忆库
- 聊天后的自动整理与索引
- 面向主流聊天程序的连接器架构

## 首版边界

### 包含

- 聊天主界面，支持输入、回复、上下文回看
- 自动生成记忆草稿，并归入速记层、事实层、主题层、档案层
- 记忆分类、标签、主题、关键词索引与搜索
- 今日概览，显示待整理记忆、最近活动、主题热度
- 主流聊天程序连接器展示与模拟收件箱同步

### 不包含

- 真正调用大模型或远端推理服务
- 真正接入 Telegram、WhatsApp、Slack、微信等官方 API
- 多人协作、账号体系、云同步、复杂权限
- 复杂提醒编排、日历深度整合

## 核心体验

### 1. 聊天即整理

用户输入一段话后，界面会同时做两件事：

1. 生成一条管家式回复
2. 判断是否值得沉淀为记忆，并生成记忆草稿

当输入带有“记住、以后、偏好、安排、整理、回顾、总结”等意图时，系统会提高记忆沉淀优先级。用户不需要跳转到复杂表单，聊天本身就是采集入口。

### 2. 分层记忆库

记忆采用四层模型：

- 速记层 `capture`：刚进入系统、等待整理的原始片段
- 事实层 `profile`：稳定偏好、人物资料、常用信息
- 主题层 `topic`：围绕健康、工作、学习、家庭、财务等主题整理过的信息
- 档案层 `archive`：阶段性总结、复盘、长期保留内容

层级体现的是信息成熟度，而不是重要性。重要性单独作为字段存储。

### 3. 连接器先统一抽象

外部消息源不直接耦合到聊天页，而是统一为连接器模型。每个连接器具备：

- 名称
- 渠道类型
- 连接状态
- 最近同步时间
- 可导入的消息摘要

首版用模拟同步展示能力边界，为后续真正接入流行聊天程序预留接口。

## 索引策略

首版使用本地轻量索引，不引入外部搜索引擎。索引由两部分组成：

- 倒排关键词索引：英文按单词切分，中文按双字切片生成 token
- 元数据索引：按层级、分类、标签、来源和更新时间过滤

搜索排序优先级：

1. 命中 token 数量
2. 重要性
3. 最近使用时间
4. 最近更新时间

## 聊天引擎策略

首版使用规则驱动的本地“管家引擎”：

- 识别输入意图
- 从记忆库召回相关上下文
- 生成一个偏生活管理风格的回复
- 视情况沉淀新的记忆草稿

这不是最终 AI 能力，而是为了先把产品结构和记忆闭环做出来。

## 界面与视觉

整体风格采用浅色自然系，不做企业后台或霓虹科技风。

### 色彩

- 背景：暖白、米白、浅雾灰
- 主强调：鼠尾草绿、柔和橄榄、温暖陶土
- 次强调：浅沙色、低饱和棕灰

### 排版与氛围

- 大留白、低压迫感
- 卡片柔和圆角，弱阴影
- 强化“生活本、管家台、回顾册”的感觉
- 聊天区更温和，记忆区更结构化

## 模块结构

- `index.html`：页面骨架
- `styles.css`：自然浅色主题、布局与组件样式
- `src/main.js`：应用启动
- `src/app.js`：界面渲染与交互编排
- `src/store.js`：本地持久化状态管理
- `src/engine.js`：聊天回复、记忆整理、主题归类
- `src/indexing.js`：token 化、索引建立、搜索
- `src/connectors.js`：连接器定义与模拟同步
- `src/data/defaults.js`：默认主题、示例对话、种子数据

## 验证策略

- 使用 Node 内置测试覆盖索引与整理逻辑
- 手动运行静态服务进行页面验证
- 手动验证聊天、记忆整理、搜索、连接器同步四条关键链路

## 风险与后续

### 已知风险

- 本地规则驱动聊天能力有限，不能替代真实大模型
- localStorage 适合原型，但不适合海量数据
- 第三方聊天程序真实接入能力依赖后续服务与授权体系

### 下一阶段

- 升级为服务端持久化
- 引入真实 LLM 聊天接口
- 接入向量检索与更强召回
- 实装至少一个真实消息程序连接器
